SOC 2 Checklist for NoSQL (MongoDB)
Organized by Trust Services Criteria. Security (CC): authentication enabled, MFA on Atlas accounts, RBAC enforced, TLS in transit, encryption at rest, audit logging, vulnerability scanning, patch management policy. Availability (A): replica sets with 3+ nodes, Atlas auto-scaling configured, backup and restore tested, RTO/RPO documented, uptime monitoring. Confidentiality (C): data classification policy, field-level encryption for PII, access reviews quarterly, data masking in non-prod. Processing Integrity (PI): change management process, schema validation rules, query performance monitoring. Privacy (P): data retention policy, deletion/anonymization procedures, GDPR/CCPA mapping. For each item: control description, evidence required for auditors, owner, and status (implemented / in progress / not applicable).
