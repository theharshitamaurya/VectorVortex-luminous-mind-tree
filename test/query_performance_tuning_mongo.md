Query Performance Tuning: MongoDB
Section 1 — Explain Plans: db.collection.explain("executionStats").find(...), key fields to read (COLLSCAN vs. IXSCAN, totalDocsExamined, executionTimeMillis, nReturned). Section 2 — Index Strategy: single field, compound (ESR rule: Equality → Sort → Range), multikey (array fields), text, partial indexes. Index anti-patterns: over-indexing (write overhead), low-cardinality fields, redundant indexes. Section 3 — Common Bottlenecks and Fixes table: COLLSCAN → add index; high docsExamined/nReturned ratio → refine index; in-memory sort → add sort field to compound index; large working set → increase RAM or shard. Section 4 — Aggregation Pipeline optimization: $match and $sort early, use covered queries, $limit before $lookup, avoid $unwind on large arrays without $match first. Section 5 — Atlas Performance Advisor: auto-suggested indexes, slow query log thresholds, real-time performance panel. Section 6 — Profiler: db.setProfilingLevel(1, { slowms: 100 }), query the system.profile collection, interpret output.
