Customer Support SOP: Database Timeout Issues
Triage flow: confirm whether timeout is connection timeout vs. operation timeout vs. pool exhaustion. Diagnostic steps: check Atlas metrics (connections, operation execution time, CPU, disk IOPS), run db.currentOp({ active: true, secs_running: { $gt: 5 } }) to find long-running operations, review slow query log (operations > 100ms), check connection pool settings in the driver config. Common causes and resolutions table: missing index (add index, use explain()), connection pool exhausted (increase maxPoolSize, check for connection leaks), CPU spike from full collection scan (add index or kill op), Atlas tier under-provisioned (scale up or enable auto-scaling), network latency between app and Atlas (check region proximity). Escalation path: L1 → L2 DBA → Atlas support ticket (include cluster ID, time range, affected operation). Customer communication templates for each severity level.
