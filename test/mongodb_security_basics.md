MongoDB Security BasicsCover authentication (enable security.authorization: enabled in mongod.conf), SCRAM-SHA-256 as the recommended auth mechanism, and x.509 for cluster auth. Explain RBAC with built-in roles (read, readWrite, dbAdmin, clusterAdmin, root) and the principle of least privilege. Cover network hardening: restrict bindIp, enable TLS (tls.mode: requireTLS), and use firewall rules to limit port 27017 exposure. Cover encryption at rest (WiredTiger Encrypted Storage Engine on Enterprise; OS-level LUKS/FileVault on Community). Include audit logging config with auditLog.destination: file and a filter for auth events. End with a hardening checklist: disable test DB, rotate credentials regularly, run MongoDB as a non-root OS user, and patch promptly.
