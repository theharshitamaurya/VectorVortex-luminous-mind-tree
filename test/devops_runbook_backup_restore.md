DevOps Runbook: Backup & Restore
Atlas section: enable continuous cloud backup in Atlas UI, set snapshot frequency and retention (hourly/daily/weekly/monthly), point-in-time restore steps via Atlas UI and Atlas CLI (atlas backups restore start), and cross-region restore procedure. Self-managed section: mongodump command with --oplog for consistent snapshots, mongorestore with --oplogReplay, filesystem snapshot approach using LVM or cloud disk snapshots (requires db.fsyncLock()). Backup verification: monthly restore drills, checksum validation, test environment restore procedure. RTO/RPO targets table. Monitoring: alert on backup job failures, track backup duration trends. Gotchas: don't restore directly to production, account for replica set lag, verify --authenticationDatabase admin flag is present.
